Plain HTML

Why plain HTML?

Because HTML is actually a pretty simple syntax to write in if you don't have to use classes. I'd prefer to write in markdown and compile that into HTML, but I haven't figured out frameworks like Jekkyl yet.

Using a classless CSS framework, you can write a site in plain HTML with little to no overhead. Because the site is so small, you don't suffer as much by linking out to a single CDN to deliver your theme.

For instance, this post is using tufte, which I found on cssbed.
